The Contract

One File. Entire Lifecycle.

Your contract.fluid.yaml is the single source of truth — logic, schema, governance, and observability.

🏗️ Metadata

Ownership, domain, data layer (Bronze/Silver/Gold), tags, and lineage context. The identity of your data product.

⚙️ Builds

Transformation steps — inline SQL, Python, dbt model references, Flink SQL, PySpark — with dependency ordering and engine selection.

📤 Exposes

Output artifacts with typed schema contracts, PII sensitivity tags (triggers auto-masking), and provider bindings that you can swap with one line.

🔒 Access Policy

RBAC grants that compile to native IAM — BigQuery Dataset roles, Snowflake GRANTs, AWS IAM policies. Declared once, enforced everywhere.

🤖 Agent Policy

Governs which AI models can access this data, for what purpose, with token limits, storage restrictions, and mandatory audit logging.

🌍 Sovereignty

Jurisdiction enforcement, data residency constraints, allowed regions. Violations are caught at plan time — before anything deploys.

fluidVersion: "0.7.1" kind: DataProduct id: analytics.customer_360 name: Customer 360 domain: analytics metadata: layer: Gold owner: team: data-platform builds: - id: transform engine: sql properties: sql: | SELECT user_id, email, SUM(ltv) FROM raw.users JOIN raw.orders USING (user_id) GROUP BY 1, 2 exposes: - exposeId: profiles binding: platform: snowflake # ← swap to gcp or aws contract: schema: - name: email sensitivity: pii # ← auto-masking accessPolicy: grants: - principal: "group:marketing" permissions: ["read"] agentPolicy: allowedModels: ["gpt-4", "claude-3"] allowedUseCases: ["analysis"] canStore: false

Governance

Built for the Agentic Era

FLUID Forge is the first data product framework with native AI agent governance.

🔒

Access Policy

Declare RBAC grants once. They compile to native IAM on every provider.

accessPolicy: grants: - principal: "group:analysts" permissions: ["read"] - principal: "role:engineer" permissions: ["read", "write"]
🤖

Agent Policy

Control which AI models can access your data, for what purpose, with token and storage limits.

agentPolicy: allowedModels: ["gpt-4", "claude-3"] deniedUseCases: ["training", "fine_tuning"] maxTokensPerDay: 1000000 canStore: false auditRequired: true
🌍

Data Sovereignty

Jurisdiction enforcement at plan time. Data never lands in a prohibited region.

sovereignty: jurisdiction: EU residency: strict allowedRegions: - europe-west1 - europe-west3 crossBorderTransfer: false

Validate

Govern

Compile Policies to Native IAM

$ fluid policy-compile contract.fluid.yaml

Translates your access grants into BigQuery roles, Snowflake GRANTs, or AWS IAM policies. Over-grant guardrails refuse OWNERSHIP-level roles.

Generate

Production Airflow DAGs

$ fluid generate-airflow contract.fluid.yaml -o dags/

Compile your contract into a native Airflow DAG. Also supports Dagster and Prefect export. Cloud Composer deployment included.

Lineage You Can See

Diagnose

System health diagnostics: Python version, provider SDK availability, config directory state, credential resolution, and feature flag status.
